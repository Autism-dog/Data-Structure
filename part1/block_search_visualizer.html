测试设计方法说明

  • 等价类划分：
    - 有效等价类：查找成功（块首/块中/块尾元素）
    - 无效等价类：查找失败（小于最小值/大于最大值/介于两块之间）
  • 边界值分析：
    - 数组边界：最小元素、最大元素
    - 索引表边界：索引表中记录的块最大值
    - 块内边界：块的第一个元素、最后一个元素

三种分块查找算法复杂度对比

算法 索引查找复杂度 块内查找复杂度 总时间复杂度 空间复杂度 本例估算（n=25，b=5，s=5）
顺序索引分块查找
block_search_sequential_index		 O(b) O(s) O(b + s) O(b)（索引表） 最多 b+s = 5+5 = 10 次比较
折半索引分块查找
block_search_binary_index		 O(log b) O(s) O(log b + s) O(b)（索引表） 最多 ⌈log₂5⌉+5 = 3+5 = 8 次比较
链式分块查找
linked_block_search		 O(b) O(s) O(b + s) O(n)（链表指针） 最多 b+s = 5+5 = 10 次比较，但需要额外 n 个链表结点

平均查找长度（ASL）计算（等概率假设）

参数说明：b = 块数（本例 5），s = 每块元素数（本例 5），n = 总元素数（本例 25）

1. 顺序索引分块查找（默认演示算法）

ASL成功 = 索引查找平均次数 + 块内查找平均次数

ASL成功 = (b+1)/2 + (s+1)/2

代入数值：(5+1)/2 + (5+1)/2 = 3 + 3 = 6

ASL不成功 = 索引查找平均次数 + 块内全遍历次数

ASL不成功 = (b+1)/2 + s

代入数值：(5+1)/2 + 5 = 3 + 5 = 8

2. 折半索引分块查找（扩展算法）

ASL成功 = ⌈log2(b+1)⌉ + (s+1)/2

代入数值：⌈log26⌉ + (5+1)/2 = 3 + 3 = 6

ASL不成功 ≈ log2(b+1) + s

代入数值：≈ 3 + 5 = 8

※ b = 块数 = √n 时平均查找性能最优。本例 b=5，s=5，总顺序查找平均次数 ≈ (b+1)/2 + (s+1)/2 = 3 + 3 = 6 次。折半索引进一步将索引查找降为 O(log b)。链式存储允许块内元素动态插删，但空间开销更大。
