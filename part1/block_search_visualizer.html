三种分块查找算法复杂度对比

算法 索引查找复杂度 块内查找复杂度 总时间复杂度 空间复杂度 本例估算（n=25，b=5，s=5）
顺序索引分块查找
block_search_sequential_index		 O(b) O(s) O(b + s) O(b)（索引表） 最多 b+s = 5+5 = 10 次比较
折半索引分块查找
block_search_binary_index		 O(log b) O(s) O(log b + s) O(b)（索引表） 最多 ⌈log₂5⌉+5 = 3+5 = 8 次比较
链式分块查找
linked_block_search		 O(b) O(s) O(b + s) O(n)（链表指针） 最多 b+s = 5+5 = 10 次比较，但需要额外 n 个链表结点

※ b = 块数 = √n 时平均查找性能最优。本例 b=5，s=5，总顺序查找平均次数 ≈ (b+1)/2 + (s+1)/2 = 3 + 3 = 6 次。折半索引进一步将索引查找降为 O(log b)。链式存储允许块内元素动态插删，但空间开销更大。
